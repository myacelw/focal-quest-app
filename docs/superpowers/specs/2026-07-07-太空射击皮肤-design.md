## 1. 目标

给训练循环加**可换肤架构** + 首个「太空射击」皮肤：孩子读方向 → 战机朝那个方向射爆陨石。核心训练机制（可调视标→报方向→翻拍→计时→结算→落库）**完全不变**，只在视标舞台套一层视觉主题 + 答对/答错/翻拍的演出，增新鲜感。

## 2. 范围

**做**
- 可换肤架构：`Skin` 接口 + 皮肤注册表（默认皮肤持久化）。
- 把现有朴素训练舞台抽成 **plain skin**（行为完全不变）。
- **space skin**：星空背景、底部战机、四方向陨石、视标 E 居中清晰、答对激光+爆炸、答错轻演出、翻拍星空流动。
- 准备页加「皮肤」选择入口，存 localStorage。
- 美术：SVG/CSS/emoji 占位，后续 AI 出图替换。

**不做**
- 不改训练机制（视标/答题/翻拍/计时/结算/落库/勋章/统计）。
- 不引入游戏引擎；陨石不做真实物理/碰撞（离散演出）。
- 打怪兽/养成皮肤（后续）。

## 3. 架构（可换肤）

**核心洞察**：把 TrainingPage 进行态的"视标舞台"（中间那块）抽象成皮肤可替换的组件，顶部（眼别+进度+静音）和底部（语音状态+触控按钮）保持通用。

```
src/skins/
  types.ts        # Skin 接口 + StageProps
  registry.ts     # SKINS 列表 + getSkinId/setSkinId(localStorage)/getSkin
  registry.test.ts
  PlainStage.tsx  # 现有朴素舞台（TumblingE + "翻！"）
  space/
    SpaceStage.tsx    # 太空射击舞台（背景+战机+陨石+视标+演出）
    Starfield.tsx     # 星空背景（CSS）
    (演出用 CSS keyframes 内联)
```

**Skin 接口**（`types.ts`）：
```ts
export interface StageProps {
  target: Direction | null            // 当前视标方向（showing 时非空）
  heightPx: number                    // 视标大小（px）
  phase: 'showing' | 'transitioning'
  lastAnswer: { dir: Direction; correct: boolean; seq: number } | null // 最近作答，seq 递增用于触发演出
}
export interface Skin {
  id: string
  name: string
  Stage: React.FC<StageProps>
}
```

**注册表**（`registry.ts`）：
- `SKINS: Skin[]`（plain、space）
- `getSkinId(): string`（localStorage `fzp.skinId`，默认 'plain'）
- `setSkinId(id: string): void`
- `getSkin(id: string): Skin`（找不到回退 plain）

**TrainingPage 改动**（最小侵入）：
- 维护 `lastAnswer` state：`handleAnswer` 里 set `{ dir, correct, seq: seq+1 }`。
- 进行态中间视标区 `<TumblingE .../>` 替换为 `<CurrentSkin.Stage target={session.target} heightPx={heightPx} phase={session.phase} lastAnswer={lastAnswer} />`。
- 当前皮肤从 `getSkin(getSkinId())` 取（准备页可切换）。
- 顶部/底部/结算/打卡等一律不动。

## 4. plain skin（行为不变）

`PlainStage`：复刻现有——`transitioning` 显示"翻！"，否则显示带 `fzpBlurIn` 动画的 `<TumblingE direction={target} heightPx={heightPx} />`。抽取后 TrainingPage 用 plain skin 渲染，视觉与现在一致（回归基线）。

## 5. space skin（太空射击）

`SpaceStage`（StageProps 驱动）：
- **背景**：`Starfield` 深色星空（CSS 渐变 + 缓慢移动的星点；`transitioning` 时加速流动表示"翻拍跃迁"）。
- **战机**：底部中央一个 SVG/emoji 战机（🚀 或简单 SVG）。
- **四方向陨石**：上/下/左/右各一颗陨石（含视标提示？不——视标单独居中）。
- **视标**：E 居中清晰显示（`heightPx`），确保"看清视标"不受演出干扰——演出在四周，视标在中心。
- **答对演出**（`lastAnswer.correct && seq 变化`）：一道激光从战机射向 `lastAnswer.dir` 方向的陨石 → 该陨石爆炸（CSS scale+fade + 火花）→ 稍后新陨石从边缘飘入。用 `useEffect([lastAnswer?.seq])` 触发一次性动画（transient state + setTimeout 清理）。
- **答错演出**：战机短暂闪红/抖动（CSS），不惩罚性。
- **翻拍过渡**（`phase==='transitioning'`）：星空加速 + 视标淡出，替代 plain 的"翻！"字（可保留一个小"跃迁"提示）。

全部用 DOM/CSS/SVG，无引擎。美术元素（战机/陨石/爆炸/激光）先用 emoji + CSS，预留后续换 AI 出图（`public/skins/space/*.png`）。

## 6. 换肤入口

准备页（`preparing` 态）加一行「皮肤」选择（下拉或按钮组：朴素 / 太空射击），选后 `setSkinId` 存 localStorage，TrainingPage 重渲染用新皮肤。默认 plain（老用户无感），用户可切 space。

## 7. 测试策略

- **纯逻辑 TDD**：`registry`——`getSkin` 默认/回退、`getSkinId`/`setSkinId` 往返（localStorage 可注入或用 jsdom；用简单函数包 localStorage，测回退与默认）。
- **手动验证**：plain 皮肤视觉与改动前一致（回归）；space 皮肤背景/战机/陨石/视标显示；答对激光+爆炸朝对应方向、答错闪、翻拍星流；准备页切皮肤生效、刷新记住；训练/结算/统计/勋章不受影响。

## 8. 拆分（供 plan 分任务）

1. Skin 接口 + registry（TDD）
2. PlainStage 抽取 + TrainingPage 接入 `Stage`（回归：视觉不变）
3. SpaceStage 静态（星空+战机+四陨石+居中视标）
4. SpaceStage 演出（答对激光爆炸/答错闪/翻拍星流）
5. 准备页换肤入口 + 走查

## 9. 默认决策（自主拍板）

1. 太空射击优先；DOM/CSS/SVG，不引擎
2. 皮肤化=套视觉层，核心训练机制不动
3. 视标居中清晰（看清优先），演出在四周
4. 默认 plain，用户在准备页可切
5. 美术先 emoji/CSS 占位，后 AI 出图
6. 演出用 `lastAnswer.seq` 递增触发一次性 CSS 动画
