## 1. 目标

给已能玩的训练循环（1a）套上"让孩子每天想来"的坚持引擎：完成训练能打卡、看到连续天数增长、攒积分，训练记录本地留存。借鉴 Duolingo 的 streak + 即时正反馈。

**成功标准**：完成一轮训练后能打卡、连续天数正确累加/中断归零、积分按规则增长、数据刷新后仍在、首页能看到打卡状态。

## 2. 范围边界

**做（1b）**
- 每日打卡（左右眼都练完 = 当天打卡，一天一次）
- 连续天数 streak（中断归零）
- 积分（每题得分 + 打卡奖励 × 连续加成系数）
- IndexedDB（Dexie）持久化：训练节记录 + 每日打卡
- 1a 训练结果**改为落库**
- 首页打卡卡片（今日状态 / 连续天数 / 累计积分）
- 训练全部完成的打卡庆祝 + 奖励音效

**不做（推迟）**
- 积分兑换 / 商城 → 迭代 2
- 勋章 / 勋章墙 → 迭代 2
- streak freeze / 补签保护 → 以后（MVP 中断归零）
- 日/周/月统计图表 → 1c
- 云同步 → 迭代 3+

## 3. 打卡逻辑

- 一次完整训练 = 左眼节 + 右眼节都完成（对应 1a 的训练节 finished）。
- 两只眼都完成时，对"今天"（本地日期 YYYY-MM-DD）打卡一次。
- 一天只打一次卡：若今天已有 checkin，重复训练不再新建 checkin（但每个训练节的记录都照常写入 `sessions`）。

## 4. 连续天数（streak）

- 打卡时计算：
  - 若最后一次打卡日期 = 昨天 → `streak = 上次streak + 1`
  - 若最后一次打卡日期 = 今天 → 已打卡，streak 不变
  - 否则（首次，或断了）→ `streak = 1`
- 显示当前 streak：若最后打卡日期是今天或昨天 → 显示其 streak；否则显示 0（已中断）。
- 纯逻辑，日期通过参数注入（不在纯函数内读时钟），保证可测；实现用 UTC 解析 YYYY-MM-DD 避免时区漂移。

## 5. 积分规则

- 每答对 1 题 = **5 分**
- 完成当日打卡额外 **+30 分**
- 连续加成系数：`coef(streak) = 1 + min(max(streak - 1, 0), 10) × 0.1`
  - streak 1 → ×1.0；streak 11 及以上 → **×2.0 封顶**
- 当日积分 = `floor((当日答对总数 × 5 + 30) × coef(streak))`
- 累计总积分 = 历次当日积分相加
- 数值为默认值，可后续调整；结构固定为"题分 + 打卡奖励，乘连续系数"。

## 6. 数据模型（Dexie / IndexedDB）

- 库名 `focalquest`，用 Dexie.js 封装。
- 表 `sessions`（每个训练节一条）：
  `{ id?: number (autoinc), date: string, startedAtMs: number, eye: 'left'|'right', answered: number, correct: number, flips: number, elapsedSec: number, acuity: number }`
- 表 `checkins`（每日一条，主键 date）：
  `{ date: string, streak: number, dailyPoints: number, totalPoints: number }`
- 索引：`sessions` 按 `date`；`checkins` 主键 `date`。
- **1a 改动**：训练节 `finished` 时写一条 `sessions`；左右眼都完成时执行打卡（读最后 checkin → 算 streak/积分 → 写/更新今天的 checkin）。

## 7. 显示

- **首页打卡卡片**：今日是否已打卡（"今天练过啦 ✓" / "今天还没练"）、当前连续天数、累计总积分。首页从 DB 读这些。
- **训练全部完成**（右眼节结束、打卡后）：庆祝画面——"打卡成功！连续 X 天 · 今日 +Y 积分"，播奖励音效。若今天已打过卡（重复训练），显示"今天已经打过卡啦"，不重复加连续天数。

## 8. 音效

- 打卡庆祝音：`sfx` 增加 `'checkin'`（比单节 `finish` 更隆重的上行琶音）。在打卡成功时播放。

**单元边界**：`date-utils`/`streak`/`points` 是无副作用纯函数（TDD）；`db` 是 Dexie 定义；`checkin` 是编排副作用（把纯函数结果写库）；UI 从 db 读并触发。

## 10. 测试策略

- **纯逻辑 TDD**：
  - `streak`：昨天打过→+1、今天已打→不变、断了→1、当前 streak 中断显示 0。
  - `points`：coef 边界（streak 1/11/20 封顶）、dailyPoints 计算与 floor。
  - `date-utils`：addDays 跨月、isYesterday、UTC 无时区漂移。
- **手动验证**：IndexedDB 存取（训练后刷新仍在）、首页打卡卡片、庆祝画面与音效、跨天 streak（可临时改注入日期测）。

## 11. 默认决策（已与用户确认）

1. 打卡 = 左右眼都练完一轮，一天一次
2. streak 中断归零（无补签保护）
3. 积分：5 分/题 + 30 打卡奖励 × 连续系数（×2 封顶）
4. 两张表 sessions / checkins
5. 1a 训练结果改为落库
6. 首页打卡卡片 + 完成庆祝 + 奖励音效
