## 1. 目标

在迭代 0（已验证：iPad Web 工程、屏幕标定、vosk 离线语音、答案映射）之上，做出**能完整玩一次的单眼训练闭环**。跑通后，孩子能独立完成"左眼 3 分钟 + 右眼 3 分钟"的一轮翻转拍训练，体验到看视标→答题→翻拍→再答的完整节奏与即时反馈。

**成功标准**：孩子能从头到尾玩完一轮训练；视标按真实物理尺寸显示；语音/触控答题灵敏；翻拍过渡顺滑；单节结算给出结果。

## 2. 范围边界

**做（1a）**
- 单眼训练节：出题 → 答题 → 判定+音效 → 翻拍过渡 → 循环 → 3 分钟计时 → 单节结算 → 换眼
- Tumbling E 视标，按"视力级别 + 40cm + 标定"换算的物理尺寸显示，随机方向
- 语音（vosk 常驻监听方向词）+ 触控方向按钮兜底
- 视标"模糊→清晰"视觉过渡，兼作翻拍窗口
- 对/错/翻拍/结算音效
- 精简训练界面（顶部只眼别 + 安静时间进度条）
- 单节结算（答对数、正确率、平均 CPM）

**不做（推迟）**
- 每日打卡、连续天数、积分 → 1b
- 训练历史持久化、日/周/月统计图表 → 1c（1a **不落数据库**，单节结果内存态）
- 打怪兽/射击等趣味关卡皮肤 → 迭代 2
- 数字视标（answer-mapping 已支持，1a 之后快速追加）
- 完整设置页 → 后续（1a 仅给最简的视力级别选择）
- 摄像头相关一切 → 迭代 3+

## 3. 训练节流程

一个"训练节" = 单眼，默认 3 分钟（可配）：

1. **准备**：提示遮住指定眼、拍子正镜片面朝眼、保持 40cm；点"开始"进入
2. **出题**：随机方向（上/下/左/右）的 E，按标定物理尺寸清晰显示
3. **答题**：孩子说方向（vosk 常驻监听）或点触控方向按钮
4. **判定 + 音效**：正确→上行音效；错误→温和下行音效（答错照常继续，记入正确率）
5. **翻拍过渡**（1.5–2 秒）：旧 E 淡出、新 E 以"模糊→清晰"淡入，同时播翻转动画 + "翻"提示音；这段即留给孩子翻拍的窗口
6. 回到第 2 步循环
7. **计时**：顶部安静的时间进度条表示 3 分钟流逝（不显秒数）
8. **单节结算**：时间到，展示答对数 / 正确率 / 平均 CPM
9. **换眼**：提示遮另一只眼，点"换眼继续"，对另一只眼重复 1–8
10. 两眼都完成 → 完成提示（1a 到此；打卡是 1b）

节奏为**自由节奏**：孩子答多快节奏就多快，每拍之间有固定的翻拍过渡窗口引导翻拍。软件**不检测翻转**（已定）。

## 4. 视标系统

- **类型**：Tumbling E（方向 E），语音答"上/下/左/右"。
- **尺寸换算**（医学标准，纯函数、TDD）：
  - 视力级别 `v`（如 0.8、1.0），视标整字张角 `α = 5 / v` 弧分（arcminute）。
  - 视标整字高度（mm）：`h = 2 × D × tan( (α 弧分转弧度) / 2 )`，`D = 400mm`。
  - 屏幕像素：`px = h × cssPxPerMm`（cssPxPerMm 来自迭代 0 标定）。
  - Tumbling E 的笔画宽 = 缺口宽 = `h / 5`（标准 5×5 网格）。
- **视力级别**：1a 提供最简选择（开始训练前选，默认 `0.8`）。说明：近距 40cm 下高视力级别视标很小（如 v=0.8 约 0.73mm），若孩子看不清可调低级别值以增大视标；具体由家长/医生定。
- 视标随机方向，避免连续重复同一方向。

## 5. 答题交互

- **语音（主）**：训练节开始时 `startVosk` 常驻监听，grammar 限定"上 下 左 右"；`parseAnswer` 解析结果 → 与当前视标方向比对 → 触发判定。识别到答案即进入判定+过渡。
- **触控（兜底，始终显示）**：底部一排四个方向按钮，孩子语音不灵时随时可点。点按等价于一次答题。
- 语音与触控走同一判定入口（`isCorrect`）。

## 6. 视觉过渡与翻拍引导

- 答题后立即进入过渡（1.5–2 秒，时长可配）：
  - 旧 E：alpha 淡出。
  - 新 E：初始 CSS `filter: blur(...)` 较大 + 低透明度 → 随时间 blur 收敛到 0、透明度到 1（模拟调节延迟，护眼防跳变）。
  - 叠加一个翻转动画图标 + "翻"提示音。
- 过渡结束，新 E 完全清晰，可答下一题。

## 7. 界面设计

参考已确认的 mockup：
- **训练界面**：顶部——只显眼别徽章 + 安静时间进度条（精简减压，不显 CPM/正确率/秒数）；中间——大 E 视标居中；底部——语音状态（"在听…"）+ 四方向触控按钮。
- **翻拍过渡**：覆盖在视标区的短暂动画（1.5–2 秒）。
- **单节结算**：居中卡片，三个指标（答对数、正确率、平均 CPM）+ "换眼继续"按钮。
- 轻美术、功能优先；趣味皮肤迭代 2。

## 8. 音效

- 正确：清脆上行音；错误：温和非惩罚下行音；翻拍："咔"提示音；结算：奖励音。
- 音量可调、可静音。素材优先免费音效库，或用 Web Audio 简单合成起步。
- 封装 `sfx` 模块，训练循环在判定/过渡/结算时调用。

## 9. CPM 计算

- **推算式**（迭代 0 已说明，依赖诚实翻拍）：每完成一次"答题 + 翻拍过渡" = 一次翻转（半 cycle）。
- `cpm = (翻转次数 / 2) / (已用分钟数)`。
- 训练中不显示，仅单节结算给平均 CPM。纯函数、TDD。

## 10. 数据

1a **不落数据库**：单节统计（答题数、正确数、翻转数、时长、眼别）为内存态，结算即时展示，训练节结束即弃。持久化（打卡、历史）留 1b 引入 IndexedDB。

## 11. 技术架构与文件结构

**复用（迭代 0）**：`src/calibration/calibration-math.ts`、`src/speech/answer-mapping.ts`、`src/speech/vosk.ts`、标定值（localStorage `fzp.cssPxPerMm`）。

**新增单元**（每个单一职责、可独立测试）：
```
src/training/
  optotype-size.ts        # 视力级别+距离+标定 → 视标毫米/像素（纯函数 TDD）
  optotype-size.test.ts
  cpm.ts                  # 推算式 CPM（纯函数 TDD）
  cpm.test.ts
  session.ts             # 训练节状态机/纯逻辑（出题、判定、计数、状态转换；TDD）
  session.test.ts
  TumblingE.tsx          # 视标渲染组件（方向 → 旋转，尺寸 → px）
  TrainingPage.tsx       # 训练界面（准备/进行/过渡/结算，集成语音+触控+音效）
  sfx.ts                 # 音效模块
```
`App.tsx` 增加进入训练页的入口（替换或新增导航项）。

**状态机（session.ts）**建议状态：`preparing` → `showing`（等答题）→ `transitioning`（翻拍过渡）→ 循环，`finished`（本节结束）。持有：当前视标方向、答题数、正确数、翻转数、剩余时间、眼别。界面通过它驱动，纯逻辑部分（出题、判定、计数、CPM）可脱离 DOM 测试。

## 12. 默认决策清单（已与用户确认）

1. 先左眼（遮右）→ 换右眼（遮左），顺序后续可配
2. 进度用安静时间进度条，不显秒数
3. 触控方向按钮始终显示
4. CPM 仅结算显示，训练中不显
5. 1a 不落库，单节结果内存态
6. 视力级别 1a 给最简选择，默认 0.8
7. 视标随机方向，答错照常继续记正确率

## 13. 测试策略

- **纯逻辑 TDD**：`optotype-size`（换算，含已知算例）、`cpm`（推算）、`session`（出题不连续重复、判定对错、计数、状态转换）。
- **手动验证**（本机 + iPad）：视标物理尺寸（尺子）、语音答题灵敏度、触控兜底、翻拍过渡观感、音效、完整走一轮左右眼。
