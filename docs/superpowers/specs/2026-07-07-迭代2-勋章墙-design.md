## 1. 目标

给已能训练+打卡的 App 加**勋章体系与勋章墙**：达成条件解锁勋章、点亮勋章墙、解锁时庆祝，给孩子持续的成就目标。30 个勋章、梯度平滑、命名萌、可成套出图。

**成功标准**：满足条件的勋章自动解锁并持久化；勋章墙展示已解锁（彩色+稀有度边框）与未解锁（置灰+进度）；训练完成时若有新解锁则庆祝。

## 2. 范围边界

**做（本迭代）**
- 30 个勋章定义（名字/emoji 占位/指标/阈值/稀有度）
- 从 sessions+checkins 算衍生指标 → 判定已达成（纯函数）
- Dexie 升级 v2 加 `badges` 表持久化解锁
- 勋章墙页（点亮/置灰 + 稀有度边框 + 未解锁进度）
- 训练完成后检测新解锁 + 庆祝弹层 + 音效
- 稀有度分级（铜/银/金/彩虹）

**不做（推迟）**
- 勋章图片素材（先 emoji 占位；后续用 AI 按"护眼小伙伴团"统一画风+稀有度边框生成，替换 emoji）
- 积分兑换/商城、彩蛋随机奖励、限时挑战、主题关卡皮肤 → 迭代 2 后续块
- 云同步 → 迭代 3+

## 4. 判定逻辑（纯函数 TDD）

- `stats-derive.ts`：`deriveStats(sessions: SessionRow[], checkins: CheckinRow[]): BadgeStats`
  - `maxStreak` = max(checkins.streak) 或 0
  - `totalSessions` = sessions.length
  - `totalSec` = Σ elapsedSec
  - `maxCpm` = max over sessions of `cpm(flips, elapsedSec)`（复用 1a `cpm`），无则 0
  - `maxAccuracy` = max over sessions where answered≥5 of `correct/answered`，无则 0
  - `totalCorrect` = Σ correct
- `evaluate.ts`：`evaluate(stats: BadgeStats): string[]` — 返回已达成的 badge id 列表（`stats[metric] >= threshold`）。
- `BADGES` 定义在 `badge-defs.ts`：`{ id, name, emoji, metric, threshold, rarity }[]`（rarity: 'bronze'|'silver'|'gold'|'rainbow'）。

## 5. 存储（Dexie v2）

- `db.ts` 升级到 `version(2)`，新增表 `badges`：主键 `id`，`{ id: string; unlockedAt: number }`。
- v1→v2 升级不改动 sessions/checkins（Dexie 自动保留）。

## 6. 解锁检测与庆祝

- `badge-service.ts`（编排）：`syncBadges(now: number): Promise<Badge[]>`
  - 读 sessions+checkins → `deriveStats` → `evaluate` 得达成 id
  - 与 `badges` 表已存的比对，找出**新解锁**的 → 写入（id + unlockedAt=now）→ 返回新解锁的勋章定义列表
- 触发点：训练全部完成打卡后调用 `syncBadges`；若返回非空 → 弹「获得新勋章」庆祝层（列出新勋章）+ 播 `badge` 音效。
- 勋章墙打开时也调一次 `syncBadges`（兜底补发）。

## 7. 勋章墙 UI

- 「勋章墙」页（导航加项）：30 个格子网格。
- 每格：已解锁 → 彩色 emoji + 名字 + 稀有度边框（铜/银/金/彩虹配色）；未解锁 → 置灰 emoji + 名字 + 进度（如"连续 5/7 天"，用 stats 对比阈值算）。
- 顶部：已解锁 X/30。
- 稀有度边框配色：铜 `#cd7f32`、银 `#9aa4ad`、金 `#e0a92c`、彩虹（渐变描边）。

## 8. 美术方向（供后续出图，本迭代先 emoji 占位）

- 统一世界观「变焦大冒险·护眼小伙伴团」；圆润萌系扁平插画（或 3D 卡通），成套统一。
- 每勋章 = 一个萌角色/形象 + 对应稀有度边框模板。
- 图片替换 emoji 时不改判定逻辑，仅换渲染资源。

## 10. 测试策略

- **纯逻辑 TDD**：`deriveStats`（各指标、answered<5 不计正确率、空输入）、`evaluate`（各 metric 阈值边界、恰好达标、未达标）。
- **手动验证**：Dexie v2 升级不丢老数据；训练达标解锁+庆祝+音效；勋章墙点亮/置灰/进度/稀有度边框；重复不重复解锁。

## 11. 默认决策（已与用户确认）

1. 30 个勋章，6 类梯度（连续/节数/时长/CPM/正确率/答对）
2. 萌形象命名 + 稀有度四级（铜/银/金/彩虹）
3. 本迭代 emoji 占位，图片后续统一画风生成替换
4. maxAccuracy 仅统计 answered≥5 的节
5. 训练完成 + 打开勋章墙都会检测补发
