# 怪兽图鉴/收集册 · 设计文档

日期：2026-07-10
状态：已获用户批准（4 个产品决策经确认，2 个设计点经呈现无异议）

## 1. 背景与目标

坚持引擎（打卡/积分/勋章/皮肤/彩蛋/连击）已就位，当前最大风险是**新鲜感消退后（第 2-4 周）的每日留存**。本功能把训练中"每题轮换怪兽"升级为宝可梦式**收集循环**：训练 → 捕获 → 图鉴点亮 → 期待明天遇到新怪。素材零成本——`public/skins/{space,shrine}/reserve/` 已有 22 只出好图的储备怪兽。

目标用户是 9-12 岁孩子；"还差 N 只没集齐"的收集欲 + "今天会抓到谁"的可变奖励，比积分数字更有驱动力。

## 2. 玩法规则（用户已确认的 4 个决策）

### 2.1 捕获机制：双轨（保底 + 彩蛋）

- **每日保底**：当天**首次完成训练打卡**时（结算流程中 `alreadyCheckedIn === false` 的那次）必得 1 只新怪兽。
- **彩蛋捕获**：彩蛋题（现有机制：`correctStreak` 达 `EGG_THRESHOLD=5` 的倍数触发，见 `src/training/session.ts`）**答对**时额外捕获 1 只，**每天至多 2 只**。
- 达到每日彩蛋捕获上限后，彩蛋题照常出现且演出不变（金光/音效/特殊庆祝），只是不再附带捕获。
- 每日上限合计 3 只（保底 1 + 彩蛋 2）。34 只全集最快约 12 天、正常约 3~4 周，与 4~6 周训练疗程契合。

### 2.2 图鉴范围：全部 34 只

- **普通 12 只**：现役太空 6 敌人 + 神庙 6 守护者（训练中每题轮换、天天见）。
- **稀有 16 只 + 史诗 6 只**：22 只储备按每世界 8 稀有 + 3 史诗分层（哪 3 只当史诗按素材观感在实现时定）。
- 每世界 17 只：太空 = 6 普 + 8 稀 + 3 史；神庙同构。

### 2.3 重复处理：不重复，只抽新怪

- 捕获时从**未拥有**的怪里抽，每次必是新怪。
- 抽取权重（按稀有度）：
  - 保底捕获：普通 70% / 稀有 25% / 史诗 5%
  - 彩蛋捕获：普通 30% / 稀有 45% / 史诗 25%（答题技巧兑换成稀有度手感）
- 某稀有度池抓空后，其权重归一化到剩余非空稀有度。
- 34 只全部集齐后停止捕获，图鉴顶部显示「🎉 全部集齐」。后续扩池只需加素材 + 定义。

### 2.4 入口：并入勋章页做 tab

- 勋章页顶部加 tab：「🏅 勋章 | 📖 图鉴」，常驻导航保持 5 项不变。
- 首页打卡卡片顺带显示收集进度（如「📖 12/34」），点击跳图鉴 tab。

## 3. 设计决策（呈现后无异议）

### 3.1 捕获揭晓时机：训练中轻提示 + 结算页开箱

- 彩蛋答对触发捕获的瞬间，只飘一个**不阻断**的小角标提示（「📖 新怪兽！」约 1.2s，皮肤无关、由训练页统一渲染）。训练中不做打断性演出——翻拍节奏是训练命脉。
- **正式揭晓在结算页**：打卡卡片下方"开箱"区，本节捕获的怪兽逐只翻牌揭晓，配音效（复用现有 `badge`/`egg` 音效，不新增素材）。
- 捕获在触发瞬间即抽定并落库（彩蛋捕获在答对时、保底捕获在打卡时），中途退出不丢怪、只是错过开箱动画。

### 3.2 皮肤池联动（进第一版）

- 已捕获的储备怪兽**加入对应皮肤的训练轮换池**：太空储备 → 太空敌人池，神庙储备 → 神庙守护者池。
- 轮换池 = 基础 6 只 + 已捕获的本世界储备怪，仍按 `seq` 循环轮换。铁律不变：视标 E 始终印在核心深色圆底上，变的是载体不是 E。
- 理由：让收集有使用价值（"抓到的怪会来训练里见你"），素材/名牌/轮换机制全是现成的。

## 4. 图鉴 UI

- 按世界分两组：🚀 太空（17）/ 🏛 神庙（17），组内按稀有度排序。
- 顶部总进度：「N/34」+ 进度条（复用 fq-* 糖果风格）。
- **已捕获**：彩色图 + 名字 + 稀有度边框（复用勋章墙稀有度边框样式）+ 捕获日期；点击弹放大卡片。
- **未捕获**：黑色剪影（CSS `filter: brightness(0)`）+「？？？」，不显示名字。
- 动效遵循现有规范（fq-rise 入场、尊重 `prefers-reduced-motion`）。

## 5. 数据与架构

### 5.1 静态定义 `src/dex/monster-defs.ts`

```ts
interface MonsterDef {
  id: string            // 如 'space-ufo'、'shrine-golem'
  world: 'space' | 'shrine'
  rarity: 'common' | 'rare' | 'epic'
  img: string           // 经 asset() 的图片路径
}
```

- 现役 12 只 id 与皮肤池 slug 对齐（`space-<enemy.name>` / `shrine-<guardian.name>`），名字翻译复用已有 `space.enemy.*` / `shrine.guardian.*` i18n key。
- 储备 22 只新增名字翻译（22 × zh/en）。图片直接引用 `reserve/` 现有路径，不搬文件。

### 5.2 捕获记录（Dexie v3）

```ts
interface MonsterRow {
  id: string          // 主键，= MonsterDef.id
  capturedAt: number  // 时间戳
  source: 'daily' | 'egg'
}
// db.version(3).stores({ monsters: 'id' })
```

- 每日彩蛋捕获数从当天记录派生（`capturedAt` 在当天 且 `source === 'egg'` 的计数），不引入额外状态。"当天"按本地日期，与打卡 `date` 字段同一套 `date-utils` 约定。

### 5.3 抽取逻辑 `src/dex/capture.ts`（纯函数，TDD）

- `pickCapture(ownedIds, source, rand)`：从未拥有中按 source 对应权重抽 1 只；池空回退；全集返回 null。
- 捕获判定（保底触发条件、彩蛋每日上限）同样以纯函数实现，副作用（落库/同步）在服务层。

### 5.4 后端双写

- server SQLite 照 badges 表模式加 `monsters` 表，双写同步 + 启动回填；后端没开不影响离线训练（复用现有降级逻辑）。
- PWA（`VITE_BACKEND=off`）下纯 IndexedDB，无后端调用。

## 6. 测试策略（延续 TDD）

- `capture.ts`：不重复抽取、权重分布（固定 rand 断言）、稀有度池空回退、每日彩蛋上限判定、全集后返回 null。
- `monster-defs.ts`：34 只完整性、id 唯一、现役 id 与皮肤 slug 对齐、每世界/每稀有度数量正确。
- Dexie v3 迁移可打开旧库。
- 结算流程：保底捕获仅在首次打卡触发、彩蛋捕获仅在彩蛋题答对且未达上限触发。
- 皮肤池联动：轮换池 = 基础 + 已捕获本世界储备的组合与轮换正确性（照现有 rotation 单测模式改）。
- i18n：新增 key zh/en 双语完整。

## 7. 第一版明确不做

- 重复捕获/升星、怪兽交换或消耗。
- 图鉴奖励加成（集齐送积分等）。
- 储备怪新出图（就用现有 22 只；扩池时再出）。
- flavor text（每只一句话小传，中英 68 条）——结构上随时可后补，不影响本版。

## 8. 风险与开放点

- **节奏风险**：若实际留存数据显示集齐太快/太慢，调每日上限或权重即可（都是常量）。
- **彩蛋演出叠加**：彩蛋庆祝 + 捕获角标同屏，注意视觉不打架（角标放角落、延迟 0.3s 出现）。
- **储备怪入池后池子变大**（最多 17 只轮换），名牌/动画机制不变，无性能顾虑（静态图）。
