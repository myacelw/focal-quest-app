# 视标尺寸优化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让训练视标能在准备页用"视标大小"滑块（带实时预览）调到孩子看得清的大小，替换掉现在按视力阈值算出的过小尺寸。

**Architecture:** 加一个 `acuityFromHeightMm` 反函数（纯函数 TDD）算等效视力级别；TrainingPage 准备页把视力级别下拉换成毫米滑块 + 实时预览 E + 等效级别标注，训练视标直接按毫米×标定显示，落库时把等效视力级别存回 `SessionRow.acuity`（不改 DB schema）。

**Tech Stack:** TypeScript · React 18 · Vitest 2 · 复用 optotype-size / TumblingE

## Global Constraints

- 滑块：**2–20mm，默认 6mm，步进 0.5**；大小存 localStorage key `fzp.optotypeSizeMm`。
- 训练视标像素：`heightPx = sizeMm × pxPerMm`（pxPerMm 来自 `fzp.cssPxPerMm` 标定）。
- 反函数（verbatim）：`rad = 2·atan(h/(2D))`；`arcMin = rad·(180·60)/π`；`acuity = 5/arcMin`，`D=400`。
- `SessionRow.acuity` 存**等效视力级别** = `acuityFromHeightMm(sizeMm)`，**不改 DB schema / 统计 / 勋章**。
- 纯逻辑 TDD；UI 手动验证。

```ts
/**
 * optotypeHeightMm 的反函数：由视标整字高度(mm)反推等效视力级别。
 * h = 2·D·tan(rad/2) ⇒ rad = 2·atan(h/(2D))；arcMin = rad·(180·60)/π；acuity = 5/arcMin。
 */
export function acuityFromHeightMm(heightMm: number, distanceMm = TRAIN_DISTANCE_MM): number {
  if (!(heightMm > 0)) {
    throw new Error(`heightMm must be > 0, got ${heightMm}`)
  }
  const rad = 2 * Math.atan(heightMm / (2 * distanceMm))
  const arcMinutes = (rad * 180 * 60) / Math.PI
  return 5 / arcMinutes
}
```
