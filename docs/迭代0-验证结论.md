# 迭代 0 技术验证结论

| 项 | 内容 |
|---|---|
| 日期 | 2026-07-06 |
| 测试设备 | 本机笔记本（Windows，Chrome + Electron 预览；DevTools 模拟 iPad Air 1180×820）。**iPad 真机未测**（手边暂无） |

## 1. 局域网 HTTPS 可达性
- 本机 `http://localhost:5173` 可正常访问、页面渲染正常、控制台无应用报错 ✅
- iPad 局域网 `https`（`npm run dev:lan`）**待真机验证**
- 备注：localhost 是浏览器安全上下文，本机 http 即可用麦克风；iPad 走局域网 IP 才需 https

## 2. 屏幕标定精度
- **逻辑链路已验证正确**：标定条 856px → 10.000 px/mm → 20mm 方块渲染成 **200×200px**（预览环境实测）✅
- **物理尺寸准确性（尺子实测 19–21mm）待 iPad 真机确认** ⏳
- 结论：算法正确，方案沿用；物理精度待真机

## 3. 语音方案 A（Web Speech API）—— ❌ 排除
| 指标 | 值 |
|---|---|
| 可用性 | API 存在，但实际识别在国内**持续 timeout** |
| 根因 | `webkitSpeechRecognition` 走 **Google 云语音服务**，国内访问不通 |
| 结论 | **国内不可用，排除** |

## 4. 语音方案 B（vosk 离线）—— ✅ 采纳
| 指标 | 值 |
|---|---|
| 本机实测 | 用户笔记本 Chrome **成功识别语音** ✅ |
| 准确率 | 主观"还行"（孩子正式量化数据待补） |
| 中文数字输出 | vosk 输出汉字（如"四"），已由 `answer-mapping.parseAnswer` 映射为数字，判定正确、不扣准确率 ✅ |
| 模型 | vosk-model-small-cn-0.22，打包为 41.8MB `.tar.gz`（需保留顶层目录；由 `scripts/prepare-vosk-model.ps1` 生成，不入库）|
| 关键坑 | vosk-browser 只认 gzip tar；官方 zip 直接用会报 "Unrecognized archive format" |
| 待补数据 | 平均延迟、模型首次加载耗时、安静环境误触发率、**孩子发音识别率（验收线 >90%）** |

## 5. 决策
- **MVP 语音方案：vosk-browser 离线**。方案 A（Web Speech）因国内 Google 不通排除。
- **标定方案：沿用**（银行卡 → px/mm → 按毫米显示视标）；物理精度待 iPad 确认。
- **升级路线**：若孩子实测识别率不足或误触发偏高，升级到 **sherpa-onnx KWS**（open-vocabulary 关键词识别，对"仅 13 个固定词"最对口）。
- **遗留 / 迭代 1 期间补验**：
  1. iPad 真机：局域网 https 可达性、标定物理尺寸（尺子量）、孩子发音识别率。
  2. vosk 模型 41.8MB 首次加载耗时在 iPad 上的体验。
  3. 安静环境误触发观察。
  4. 交互：语音为主 + 触控兜底（已定），iPad 上确认孩子单手操作可行性。
