# 🚀 变焦大冒险 / FocalQuest

> 把儿童"翻转拍"视力训练，从枯燥的读卡片，变成有节奏、音效、打卡积分的 **iPad 游戏**。

一个给自家 9–12 岁孩子用的视力训练游戏化 App。核心是解决一个很具体的痛点：**孩子有训练拍子，但太无聊、坚持不下来**。

---

## 📖 这是什么？（零背景 3 分钟看懂）

### 什么是"翻转拍"训练？
"翻转拍"（学名 *Accommodative Flipper*，调节灵敏度训练）是眼科/视光常用的一种家庭视力训练工具——一个手柄上装了两组镜片（一面 +2.00D 正镜片、一面 −2.00D 负镜片）。训练时：

1. 遮住一只眼，透过**正镜片**看远处一个小字/符号，努力看清；
2. 看清后把拍子**翻转**过来，透过**负镜片**再看清同一个目标；
3. 如此反复翻转，锻炼眼睛"变焦"（调节）的灵敏度。

一次"看清正镜片 + 看清负镜片"叫 **1 个 cycle**，每分钟完成的 cycle 数（**CPM**, Cycles Per Minute）是衡量训练效果的核心指标。

### 痛点在哪？
传统训练就是对着一张卡片一遍遍读，**极其枯燥**，孩子坚持几天就放弃。买了拍子吃灰是常态。

### 这个 App 怎么解决？
把训练包装成游戏：

- **视标**（要看清的目标）不再是死板的卡片，而是印在**会动的怪兽/敌人**身上（太空战机、神庙守护者……）；
- 孩子看清后**用语音报出方向**（"上/下/左/右"）或点屏幕作答，答对有**音效 + 爆炸/击碎特效**；
- 有 **每日打卡、连续天数、积分、勋章墙、皮肤解锁、随机彩蛋**——一整套"坚持引擎"，让孩子每天想练。

> ⚠️ **诚实边界**：这是**家庭级**工具，不是临床级。软件用节奏引导孩子翻拍，但**不检测**他是否真的翻了拍（不做摄像头识别），有效性依赖孩子诚实训练 + 家长偶尔监督。CPM 是推算值。

## ✨ 功能一览（已实现）

| 模块 | 功能 |
|---|---|
| **核心训练循环** | Tumbling E 视标、语音（离线）+ 触控 + 键盘（方向键/1-4/asdf/jkl;）三通道答题、翻拍过渡动画、计时换眼、单节结算 |
| **坚持引擎** | 每日打卡、连续天数、积分、完成庆祝 + 音效 |
| **统计** | 日/周/月的 CPM / 正确率 / 训练次数图表（手绘 SVG）+ 家长周报建议 |
| **勋章墙** | 30 枚勋章（6 类梯度、萌系命名、四级稀有度，AI 出图），从训练记录自动判定解锁 |
| **可换肤** | 朴素 / 太空射击 / 神庙勇者 三套皮肤（可扩展架构）；每题轮换不同怪兽/敌人 |
| **积分解锁** | 练习赚分达门槛永久解锁新皮肤，首页/设置页显示解锁进度 |
| **随机彩蛋 / 连击** | 连对 5 个 → 彩蛋（金光/宝箱）；连对 3+ 飘「🔥 连击 ×N」（可无限累加） |
| **极简路径** | 首页大按钮 → 遮眼提示 1.5s 自动进训练；孩子只需"点一下 → 答方向"，配置全在家长设置页 |
| **翻拍引导** | 每次翻拍统一醒目「🔄 翻转拍子」+ 节奏进度条 + 咔哒音（训练命脉），速度快/适中/慢可调 |
| **暂停 / 时长** | 训练可暂停续、剩余时间倒计时可见、单眼时长 1/2/3/5 分可配 |
| **屏幕标定** | 用银行卡（85.6mm）校准像素/毫米，保证视标物理尺寸准确 |
| **专业指导** | 设置页记录拍子度数（±2.00 等）、视标专业指导、「关于训练·家长必读」，诚实的家庭级定位 |
| **中英双语** | 按浏览器语言自动选择，设置页可手动切换（🌐 语言） |
| **离线 PWA** | 纯前端可部署 GitHub Pages，装到 iPad 主屏幕后完全离线可用（含语音识别） |
| **数据** | 本地 IndexedDB 为可靠源；本机开发时可选双写同步到 Node+SQLite 后端（防清缓存/换设备丢数据） |

界面为**风格 B「糖果能量」**：明亮渐变、圆润彩色卡片、图标胶囊导航（孩子挑的）。

## 🏗️ 技术架构

App 有两种运行形态，代码完全一致，只是构建参数不同：

```
┌───────────────────── React + TypeScript + Vite 前端 ─────────────────────┐
│  ├─ 视标/动画/皮肤：DOM + CSS（无游戏引擎）                                │
│  ├─ 语音：vosk-browser 离线识别（国内可用，不依赖云）                      │
│  ├─ 本地存储：IndexedDB（Dexie.js）── 可靠数据源、离线可用                 │
│  ├─ i18n：轻量零依赖中英双语（src/i18n.tsx）                              │
│  └─ 全局视觉系统：src/index.css（风格 B）                                 │
└───────────────────────────────┬───────────────────────────────────────────┘
                                 │
              ┌──────────────────┴──────────────────┐
              ▼                                      ▼
   ① 部署模式（GitHub Pages）              ② 本机开发模式（npm start）
   纯静态 PWA，VITE_BACKEND=off             同时跑 Vite + 本机 Node 后端
   自定义 Service Worker（src/sw.ts）        /api（Vite proxy 转发，免 CORS）
   ├─ 补 COOP/COEP 头 → vosk 可加载                    ▼
   │  （GitHub Pages 本身不能设响应头）      本机 Node 后端（server/）
   ├─ 离线预缓存首屏 + 模型                  ├─ Node 24 内置 node:sqlite
   └─ 装到主屏幕 = 离线 App                  └─ 前端 best-effort 双写同步
```

**部署在哪？** [GitHub Pages](https://myacelw.github.io/focal-quest-app/)（免费静态托管）。iPad Safari 打开→添加到主屏幕，首次联网装好后完全离线可用，语音也能用（跨域隔离头由 Service Worker 补齐）。详见 [docs/部署到-GitHub-Pages.md](docs/部署到-GitHub-Pages.md)。

**为什么不用 Vercel / Supabase / EdgeOne？** 已实测：`*.vercel.app` 在国内被 DNS 污染 + SNI 封锁；Supabase 无国内节点；EdgeOne Pages 免备案的默认域名对中国大陆网络直接 401（只有绑定备案域名才有大陆节点）。GitHub Pages 同样没有大陆节点，但装成 PWA 后只有"首次安装 + 每次更新"需要联网、且更新只拉几十 KB，把"慢"摊薄成一次性代价，可接受。

**为什么语音用 vosk 而非浏览器自带？** Web Speech API 走 Google 云，国内不通，已排除。vosk 是离线识别，国内可用。

## 🚀 快速开始

### 最快：直接在 iPad 上玩
打开 **https://myacelw.github.io/focal-quest-app/** → Safari 分享 → 添加到主屏幕。
首次联网装好后可离线使用，无需任何本机环境。

### 想本地开发 / 改代码

#### 前置
- Node.js **24+**（用到内置 `node:sqlite` 和原生 TS 运行）
- 一台 iPad 和电脑在**同一 WiFi**（iPad 实机训练用）

### 安装
```bash
npm install
```

### 准备语音模型（首次，约 42MB，不入库）
```bash
pwsh scripts/prepare-vosk-model.ps1
```

### 开发运行

**本机浏览器测试**（localhost 即安全上下文，麦克风可用）：
```bash
npm run dev          # → http://localhost:5173
```

**iPad 实机训练** —— 一键启动（推荐）：
```bash
npm start            # 同时拉起 后端 + Vite(局域网 https)，打印 iPad 访问地址；Ctrl+C 一起停
```
然后 iPad 打开脚本打印的 `https://<电脑IP>:5173`（首次需信任自签证书）。脚本会自动
清理上次残留、选对局域网 IP、Ctrl+C 时把两个服务一起干净停掉。

> 想分开跑也行：`npm run server`（后端 :3001）+ `npm run dev:lan`（Vite）各开一个终端。

### 首次使用流程
1. 首次打开有 5 步引导，教怎么正确训练；
2. 进「⚙️ 设置 → 📐 屏幕标定」：把屏幕上的条拖到和一张银行卡一样宽 → 保存（只需一次，视标物理尺寸才准）；
3. 家长在「⚙️ 设置」配一次：视标大小 / 单眼时长 / 翻拍速度 / 皮肤 / 拍子度数 / 语言（都有默认值，可先跳过）；
4. 回首页点「▶ 开始今日训练」→ 遮眼提示 1.5s 后自动进训练（孩子不用再点）；
5. 遮一只眼、透过拍子看清视标 → 报方向 / 点按钮 / 按键盘 → 跟「🔄 翻转拍子」提示翻拍 → 循环；
6. 每眼练完自动换眼，两眼完成 → 打卡。

### 测试
```bash
npm test             # 124 单元测试
```

## 🧭 关键设计决策（反复讨论定下的边界）

1. **只做 iPad Web**——不做手机/电脑/原生 App/Flutter。
2. **语音报答案为主 + 触控/键盘兜底**——不做手势；语音识别范围仅方向上下左右（iPad 主要靠语音+触控，键盘方便电脑调试）。
3. **节奏引导，不检测翻转**——不做任何摄像头 CV。代价：家庭级够用，非临床级。
4. **数据本地优先**（IndexedDB 是唯一可靠源）；本机开发时可选双写到 Node+SQLite 后端，部署版不带后端——不做云同步/账号（远期）。
5. **MVP 轻美术，重坚持引擎**——不引入游戏引擎（Pixi/Phaser）。
6. **屏幕物理标定必做**——否则视标物理尺寸无意义。

  - ✅ 纯前端 PWA 改造，部署 GitHub Pages（离线可用、装主屏幕）
  - ✅ 中英双语国际化（按浏览器语言自动选择）

### 通用"变形框架"（皮肤扩展思路）
每个皮肤 = **怪兽池 / 英雄池 / 场景池 / 特效池** 四个可换的槽，换槽不用改引擎。同一套训练循环，可以是太空歼灭战、神庙勇者、植物大战僵尸、宝可梦对战……只是叙事和素材不同。
